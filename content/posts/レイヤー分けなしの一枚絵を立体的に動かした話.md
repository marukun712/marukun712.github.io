# どうやって動いている?
このデモサイトは、自作ライブラリの`@kokoro/rig`で動いています。

このライブラリは、"エンジニアのための2Dキャラクターメッシュ変形ライブラリ"を目標に、既存の2Dキャラクターメッシュ変形ツールで必要となっていた複雑な概念・GUI等の学習コスト無しで、エンジニア、さらにLLMがコーディングのみでキャラクターの動作を直感的に記述することを目指しています。

## 基本的な概念
このライブラリにおいて、キャラクターを動かすために必要な概念は一つだけで、`(u, v) => 変形`という関数のみです。この関数をポーズ関数と呼びます。

引数として取られている(u, v)は、変形対象の頂点のUV座標、つまり左上を0,0、右下を(1,1)とした座標です。

変形量としては、`x`,`y`だけでなく`rot(回転)`や`pivot(回転の中心)`も指定できます。

例えば、体を左向きに傾ける`left`というポーズは、以下のような関数で定義されます。
```typescript
left: (u: number, v: number) => {
    const { fromTop } = getSpatialParams(u, v);
    const w = curve.power2(fromTop);

    return {
        tx: -200 * w,
        ty: 0,
    };
},
```

UV座標が引数として取れることで、相対的な頂点の位置に基づいた重み付けをすることができます。
この実装例で言えば、UV座標から算出される`fromTop`(対象となる画像の一番上を1,一番下を0とした値)を2乗の曲線に渡し、その値を移動量`-200`にかけることで、上半身ほど大きく動き、下半身はあまり動かないという自然な動きをたった2行で記述できます。

もし、胴体や胸だけを動かしたい場合は、`sin`などの山形の曲線を用いて、同じようにUV座標から重みを求めればよいです。

## 深度推定との組み合わせ

ここからが、このデモの核心部分となります。先ほどの基本的な概念を用いて、キャラクターを斜めに傾けたり、一部分を移動させる、といったことは可能ですが、立体表現をするために必要な複雑な変形を表現する曲線の組み合わせを探すことは困難です。

そこで、視差の考え方を導入します。つまり、手前のものは大きく動き、遠くのものはあまり動かないということです。先ほどUV座標から曲線を使って上半身ほど大きく動く動きを表現したように、UV座標から手前/奥という情報を判別できれば、移動量をスケールして頂点単位の視差を作り出すことができそうです。

ここで登場するのが、深度推定モデルです。

https://github.com/marukun712/kokoro/blob/master/src/depth.ts

デモのソースコードからわかるように、`getDepth`関数を使って、UV座標から奥行きを取得するための`getDepthFromUV`関数と、深度データ、width、heightを取得できます。

この`getDepthFromUV`関数を`DEPTH_TEMPLATE`に渡し、80という移動量を奥行きに応じて自動でスケールするポーズを生成しています。

https://github.com/marukun712/kokoro/blob/master/packages/rig/src/depth/index.ts

https://github.com/marukun712/kokoro/blob/master/packages/rig/src/depth/worker.ts

こちらがライブラリ内部のソースコードです。ブラウザで動作する深度推定モデルとして、`depth-anything-v2`を使用しています。また、UIが固まらないようWorkerに逃がして実行しています。

従来、立ち絵を動かすときに用いる視差といえば、レイヤーベースで、前髪は大きく動き、後ろ髪はあまり動かない、のような形でした。ですが、この手法はレイヤー分けされていない一枚絵でも頂点単位で深度を自動推定するため、より多くの素材を立体的に動かすことができます。
